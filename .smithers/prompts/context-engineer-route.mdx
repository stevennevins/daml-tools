## Choose a route

Pick exactly one `selectedRoute`:

- `single_task` — one worker agent can finish this in a single pass.
- `skills` — the work is best done by a worker agent loading specific skills.
- `workflow` — the work earns a durable seeded workflow (ordering, loops,
  approvals, crash-recovery).
- `manual` — the work should be handed to a human, not executed automatically.

Then fill in:

- `selectedSkills` — skills the executor should load, best-first (lean on the
  contract's `availableSkills`).
- `selectedWorkflow` — if `selectedRoute` is `workflow`, the single best-fit id
  from the list above; otherwise `null`.
- `durableRequired` — true when the route genuinely needs durable execution.
- `humanApprovalRequired` — true when a human gate is needed before side effects
  (lean on the contract's `risks` and any side-effecting `desiredArtifacts`).
- `reason` — why this route fits the contract and classification.

Be decisive and concrete. The next steps add backpressure gates and then execute
along this route.
