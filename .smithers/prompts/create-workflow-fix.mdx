# Fix the workflow until its graph renders

The new workflow failed to render its graph. `smithers graph` loads the module
(catching TypeScript / import / JSX errors) and renders one frame without
executing it (catching structural errors). Read the error, fix the files, and
make it load. You have file + shell tools.

## How to fix

1. Read the entry file and any prompt/component files it imports.
2. Map the error to a cause. Common ones:
   - **Import / module not found** — a prompt or component path is wrong, or a
     named export doesn't exist. Check `{props.promptsDir}` and `../components`.
   - **Type / schema error** — a `<Task output={outputs.x}>` doesn't match its
     Zod schema, or a destructured component (`Branch`, `Loop`, `Approval`) was
     never pulled out of `createSmithers(...)`.
   - **JSX / structural** — a conditional renders the wrong child type, a node id
     is duplicated, or a control-flow component is missing a required prop
     (`Loop` needs `until`; `Approval` needs `request`).
3. Make the smallest change that fixes the error without abandoning the design.
   If the docs would help, run `bunx smithers-orchestrator docs` or
   `bunx smithers-orchestrator ask "<question>"`.
4. Re-run `bunx smithers-orchestrator graph {props.workflowFile}` yourself to
   confirm it now exits 0 before you finish.

Do not delete the workflow or stub it out to "pass" — fix the real problem.
