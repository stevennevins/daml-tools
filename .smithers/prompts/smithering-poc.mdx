in artifacts/smithering/poc/ (never merged; reference only). Read docs/planning/03-eng.md
and docs/planning/05-tickets.md. Goal: surface what the plan got wrong while it is still
cheap.

Report learnings[] and engDocAmendments (or null). Update docs/planning/03-eng.md if
needed — and if learnings invalidate, reorder, or add tickets, update
docs/planning/05-tickets.md AND artifacts/smithering/tickets.json too (keep existing
ticket ids stable; add new tickets rather than renaming). The tickets.json file, not the
earlier task output, is the contract the implementation workflow imports. ALWAYS return a
result; a POC that failed is itself a finding.
